GAMBAR 2

Flowchart Alur Pelanggan (Ref: image_c5961c.png)

Mulai: Landing Page
DECISION GATE

Pilih Jenis Layanan?

A. Pemesanan Unit Baru
Langkah A1

Memilih Unit Mobil Status 'Available'

Langkah A2

Isi Formulir Booking & Kontak Customer

Langkah A3 - RDBMS

Eksekusi DB::transaction & lockForUpdate

Langkah A4

Sistem Menerbitkan Invoice Code (INV-)

B. Pelacakan & Pelunasan
Langkah B1

Masukkan Nomor Telepon Customer

Langkah B2

Lihat Riwayat Invoice Status 'Unpaid'

Langkah B3

Unggah Bukti Transfer Resi Fisik Asli

Langkah B4 - Menunggu

Status Berubah: Pending Validation

DECISION BLOCK

Apakah Bukti Pembayaran Sah / Valid?

❌ Tidak (Tolak) ✓ Ya (Approve)
Selesai: Serah Terima Unit (Status Paid & Sold)
Mermaid.js Code 
flowchart TD
    Start([Mulai: Landing Page]) --> ServiceDecision{Pilih Menu Layanan?}

    %% Cabang A: Booking Unit Baru
    ServiceDecision -->|A: Pemesanan Unit| SelectUnit[Pilih Unit Mobil Status 'Available']
    SelectUnit --> FillForm[Isi Formulir Booking & Kontak Customer]
    FillForm --> DBTrans[Eksekusi DB::transaction & lockForUpdate]
    DBTrans --> GenerateInv[Terbitkan Invoice Unik & Status 'Unpaid']
    GenerateInv --> UploadReceiptPrompt[Arahkan ke Menu Lacak untuk Unggah Bukti]

    %% Cabang B: Lacak & Pelunasan
    ServiceDecision -->|B: Pelacakan & Pelunasan| TrackInvoice[Masukkan No HP Customer]
    TrackInvoice --> SelectInvoice[Pilih Invoice Status 'Unpaid']
    SelectInvoice --> UploadReceipt[Unggah Bukti Transfer Resi Fisik Asli]
    UploadReceipt --> StatusPending[Status Invoice Berubah: Pending Validation]
    StatusPending --> WaitValidation[Menunggu Verifikasi Kasir]

    UploadReceiptPrompt --> TrackInvoice
    WaitValidation --> VerifyDecision{Bukti Transfer Sah?}
    VerifyDecision -->|Tidak| TolakBukti[Tolak: Status Tetap Pending / Tolak Pembayaran]
    VerifyDecision -->|Ya| SahkanLunas[Sahkan Lunas: Status Paid & Unit Sold]

    TolakBukti --> TrackInvoice
    SahkanLunas --> PrintReceipt[Terbitkan Kwitansi Resmi Lunas]
    PrintReceipt --> End([Selesai: Penyerahan Unit])

Flowchart ini diadaptasi dari struktur visual komparatif image_c5961c.png untuk kejelasan pembaca.

GAMBAR 3

Flowchart Alur Verifikasi Kasir (Ref: image_c59621.png)

Start: Portal Showroom
SECURITY BY OBSCURITY

Sistem Memicu Kombinasi: Ctrl + Shift + A

Login Kredensial Kasir & Masuk Dashboard

DECISION GATE

Pilih Menu Operasional Kasir?

A. Validasi Pembayaran
Proses A1

Tinjau Invoice Status 'Pending Validation'

Proses A2

Buka Berkas Bukti Resi Fisik Asli

Decision A3

Resi Valid & Sesuai?

Proses A4 - DB Action

Sahkan Lunas: Status Paid & Item Sold

B. Manajemen Inventaris
Proses B1

Buka Menu Kelola Unit Mobil Premium

Proses B2

Pilih Aksi: Simpan, Ubah, atau Hapus

Decision B3

Apakah Unit Terikat Transaksi Aktif?

Proses B4 - DB Restriction

Gagal Hapus: ON DELETE RESTRICT

Selesai: Kembali ke Dashboard
Mermaid.js Code 
flowchart TD
    Start([Mulai]) --> Login[Login Portal Tersembunyi: Ctrl + Shift + A]
    Login --> Dashboard[Masuk Dashboard Control Panel]
    Dashboard --> ActionDecision{Pilih Menu Operasional?}

    %% Cabang A: Verifikasi Pembayaran
    ActionDecision -->|A: Validasi Pembayaran| CheckInvoice[Tinjau Invoice Status 'Pending Validation']
    CheckInvoice --> OpenReceipt[Buka Berkas Bukti Resi Fisik Asli]
    OpenReceipt --> ApprovalDecision{Bukti Transfer Valid?}
    ApprovalDecision -->|Tidak| RejectPayment[Tolak Bukti & Kirim Notifikasi Tolak]
    ApprovalDecision -->|Ya| ApprovePayment[Sahkan Lunas: Hubungkan ID Kasir]
    ApprovePayment --> MutateStatus[Mutasi Invoice ke 'Paid' & Item ke 'Sold']
    MutateStatus --> PrintInvoice[Cetak Struk/Kwitansi Lunas Resmi]

    %% Cabang B: Manajemen Inventaris (CRUD)
    ActionDecision -->|B: Manajemen Inventaris| MenuItems[Buka Menu Kelola Unit Mobil]
    MenuItems --> SelectAction{Pilih Aksi?}
    SelectAction -->|Create/Update| SaveItem[Validasi Spesifikasi & Simpan Unit]
    SelectAction -->|Delete| DeleteItem[Eksekusi Hapus Unit]
    DeleteItem --> ConstraintDecision{Apakah Unit Memiliki Invoice Aktif?}
    ConstraintDecision -->|Ya| RestrictAction[Tolak Penghapusan: ON DELETE RESTRICT]
    ConstraintDecision -->|Tidak| ConfirmDelete[Hapus Data Unit dari Database]

    RejectPayment --> Dashboard
    PrintInvoice --> Dashboard
    SaveItem --> Dashboard
    RestrictAction --> Dashboard
    ConfirmDelete --> Dashboard

id : bigint (PK)

item_id : bigint (FK) - RESTRICT
